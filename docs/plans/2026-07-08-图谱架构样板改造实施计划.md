# 图谱架构样板改造实施计划

> **For agentic workers:** REQUIRED SUB-SKILL: Use inline execution task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 将 Graph 浏览模块改造成前端 `features/graph` + 后端 `modules/graph` 的架构样板，并保持现有图谱交互行为。

**Architecture:** 前端 `app/graph/page.tsx` 只保留路由入口和 Suspense，图谱状态、API 和组合 UI 进入 `features/graph`。后端 Graph 使用 `modules/graph/dependencies/service/router/schemas` 包装现有 `models.repository.GraphRepository`，路由返回统一响应。Chat 只复用新的 repository dependency，不改 RAG 业务逻辑。

**Tech Stack:** Next.js 16、React 19、TypeScript strict、FastAPI、Pydantic v2、Neo4j、pytest、pnpm。

## Global Constraints

- 文档名使用日期 + 中文说明。
- 不改用户未提交的 `Make.ps1`、`start.ps1` 和新增架构文档。
- 不改 Neo4j 查询语义和 D3 画布布局算法。
- 不迁移 Learn、Labs、CMS。
- 不重写 Chat RAG 生成逻辑。
- 前端 Graph 业务代码进入 `frontend/src/features/graph`。
- 后端 Graph 业务代码进入 `backend/modules/graph`。
- 项目内 Graph 页面不再直接依赖 `frontend/src/lib/api.ts`。
- `frontend/src/lib/api.ts` 暂时保留为兼容 re-export，避免 Learn 的 `MiniGraph` 连带大改。
