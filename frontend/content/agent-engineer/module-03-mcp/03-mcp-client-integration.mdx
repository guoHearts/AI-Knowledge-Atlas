# MCP Client 集成到 Agent

> 💡 **Agent 自动发现 MCP 工具** — 像给程序员配了一套标准工具箱，不用每次告诉它工具有什么。

## MCP Client 的角色

```
┌──────────┐     ┌──────────────┐     ┌──────────────┐
│  Agent   │────▶│ MCP Client   │────▶│ MCP Server A │
│          │     │              │────▶│ MCP Server B │
│          │     │ 管理所有连接  │────▶│ MCP Server C │
└──────────┘     └──────────────┘     └──────────────┘
                       │
                  ┌────▼────┐
                  │ Tool 注册表│
                  │ 自动发现  │
                  │ 路由分发  │
                  └─────────┘
```

MCP Client 承担三个关键职责：
1. **连接管理**：与所有 MCP Server 建立和维护连接
2. **工具发现**：自动收集所有 Server 提供的 Tool
3. **路由分发**：将 Agent 的 tool_use 请求路由到正确的 Server

## 方案一：mcp-agent（Python 首选）

参考 [lastmile-ai/mcp-agent](https://github.com/lastmile-ai/mcp-agent)（8.3k stars）：

```python
from mcp_agent import Agent, MCPApp

app = MCPApp("my-agent-app")

# 连接多个 MCP Server
@app.mcp_server("weather", transport="http", url="http://localhost:3001")
@app.mcp_server("database", transport="http", url="http://localhost:3002")
@app.mcp_server("email", transport="http", url="http://localhost:3003")

async def main():
    agent = Agent(
        model="claude-sonnet-4-6",
        instruction="你是全能助手，能查天气、查数据、发邮件。",
        servers=["weather", "database", "email"],  # 连接这 3 个 Server
    )

    # Agent 自动发现所有 3 个 Server 的工具
    response = await agent.send_message(
        "查下北京天气，如果下雨就发邮件通知团队明天改线上开会"
    )
    # Agent 自动：查天气 → 判断是否下雨 → 发邮件
    print(response)

if __name__ == "__main__":
    import asyncio
    asyncio.run(main())
```

## 方案二：TypeScript 手动集成

```typescript
import { Client } from '@modelcontextprotocol/sdk/client/index.js';
import { StdioClientTransport } from '@modelcontextprotocol/sdk/client/stdio.js';

// 步骤 1：创建 MCP Client 连接
const transport = new StdioClientTransport({
  command: 'python',
  args: ['server.py'],
});

const client = new Client({
  name: 'my-agent-client',
  version: '1.0.0',
});

await client.connect(transport);

// 步骤 2：获取所有可用 Tool
const { tools } = await client.listTools();
console.log('可用工具:', tools.map(t => t.name));
// → ['get_weather', 'send_email', 'search_web']

// 步骤 3：将工具传给 Agent
import { Agent } from '@anthropic-ai/sdk';

const agent = new Agent({
  model: 'claude-sonnet-4-6',
  systemPrompt: '你是全能助手，根据需要使用提供的工具。',
  tools: tools.map(t => ({
    name: t.name,
    description: t.description,
    parameters: t.inputSchema,
  })),
});

// 步骤 4：Agent 调用 Tool 时，通过 MCP Client 执行
const response = await agent.sendMessage('查北京天气');
// Agent 返回 tool_use → 你的代码转发给 MCP Server

if (response.toolUse) {
  const result = await client.callTool({
    name: response.toolUse.name,
    arguments: response.toolUse.input,
  });
  // 将结果返回给 Agent
}
```

## 多 Server 路由策略

当有多个 MCP Server 时，Agent 需要选择合适的 Tool：

```
用户: "查下 Q2 销售数据，然后发邮件给老板"

Agent 判断:
├── "查数据" → query_database Tool (database MCP Server)
└── "发邮件" → send_email Tool (email MCP Server)

自动路由，无需人工干预
```

## MCP Client 最佳实践

### 1. 工具去重

```typescript
// 多个 Server 可能提供同名 Tool
const allTools = new Map<string, Tool>();

for (const server of connectedServers) {
  const { tools } = await server.listTools();
  for (const tool of tools) {
    if (allTools.has(tool.name)) {
      console.warn(`⚠️ Tool "${tool.name}" 冲突，使用先注册的`);
      continue;
    }
    allTools.set(tool.name, tool);
  }
}
```

### 2. 错误重试

```typescript
async function callToolWithRetry(
  client: Client,
  toolName: string,
  args: Record<string, unknown>,
  maxRetries = 3
): Promise<any> {
  for (let i = 0; i < maxRetries; i++) {
    try {
      return await client.callTool({ name: toolName, arguments: args });
    } catch (error: any) {
      if (i === maxRetries - 1) throw error;
      console.warn(`Tool ${toolName} 失败，重试 ${i + 1}/${maxRetries}`);
      await new Promise(r => setTimeout(r, 1000 * (i + 1)));
    }
  }
}
```

### 3. 超时控制

```typescript
async function callToolWithTimeout(
  client: Client,
  toolName: string,
  args: Record<string, unknown>,
  timeoutMs = 30000
): Promise<any> {
  return Promise.race([
    client.callTool({ name: toolName, arguments: args }),
    new Promise((_, reject) =>
      setTimeout(() => reject(new Error(`Tool ${toolName} 超时 (${timeoutMs}ms)`)), timeoutMs)
    ),
  ]);
}
```
