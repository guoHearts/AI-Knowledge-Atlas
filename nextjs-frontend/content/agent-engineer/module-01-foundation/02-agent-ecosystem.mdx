适合需要深度定制的开发者。

### 💡 选择建议

```python
# 如果你想快速上手 → LangChain
from langchain.agents import create_openai_functions_agent
agent = create_openai_functions_agent(llm, tools, prompt)

# 如果你需要复杂工作流 → LangGraph
from langgraph.graph import StateGraph
graph = StateGraph(MyState)
graph.add_node("analyze", analyze_node)
graph.add_node("execute", execute_node)
graph.add_edge("analyze", "execute")

# 如果你做多 Agent 协作 → CrewAI
from crewai import Agent, Task, Crew
researcher = Agent(role="研究员", goal="搜索相关技术资料")
writer = Agent(role="技术作家", goal="输出技术文档")
crew = Crew(agents=[researcher, writer], tasks=[...])
```

## 赛道二：协议/标准 — 连接一切

协议是 Agent 生态的"USB 标准"。目前最核心的是 **MCP（Model Context Protocol）**。

### 主要协议

| 协议 | 提出者 | 用途 | 状态 |
|------|--------|------|------|
| **MCP** | Anthropic | Agent ↔ 工具连接 | ✅ 广泛采用 |
| **A2A** | Google | Agent ↔ Agent 通信 | 🆕 2025 发布 |
| **AG-UI** | CopilotKit | Agent ↔ 前端 UI | 🔥 快速增长 |
| **UTCP** | 社区 | 通用工具调用协议 | 📋 提案阶段 |

### MCP 为什么重要？

```
传统方式：每个工具单独集成
Agent ──→ 天气 API（专用代码）
Agent ──→ 数据库（专用代码）
Agent ──→ 邮件服务（专用代码）

MCP 方式：一次连接，所有工具可用
Agent ──→ MCP Client ──→ MCP Server 1（天气）
                      ──→ MCP Server 2（数据库）
                      ──→ MCP Server 3（邮件）
```

参考实现：[PrefectHQ/fastmcp](https://github.com/PrefectHQ/fastmcp) — 25k+ stars，Pythonic MCP 开发框架

## 赛道三：平台/低代码 — 最易上手

适合不写代码或快速原型验证。

| 平台 | 特点 | 适合 |
|------|------|------|
| **Dify** | 开源，可视化工作流 | 快速搭建 AI 应用 |
| **Coze (扣子)** | 字节跳动，免费 | 个人开发者/学生 |
| **FastGPT** | 开源，知识库强 | 企业知识问答 |
| **n8n** | 工作流自动化 | 集成多种服务 |

## 2026 年趋势

1. **MCP 成为标配**：几乎所有主流 Agent 框架都支持 MCP
2. **多模态 Agent**：视觉+音频+语言的统一感知（如 Nemotron Nano Omni）
3. **Computer Use**：Agent 直接操作电脑（Claude Computer Use、Browser Use）
4. **从 Demo 到生产**：可观测性、安全护栏、部署方案日趋成熟
