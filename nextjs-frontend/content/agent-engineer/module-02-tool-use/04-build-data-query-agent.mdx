# 实战——构建一个数据查询 Agent

> 💡 **让 Agent 自己写 SQL** — 自然语言 → SQL 生成 → 自动执行 → 结果解释。

## 目标

构建一个能用自然语言查询数据库的 Agent，它会：
1. 理解用户的问题
2. 自动生成正确的 SQL
3. 执行查询
4. 将结果转化为用户能理解的回答

## 整体架构

```
用户: "工程部有哪些员工？"
        │
        ▼
┌───────────────────┐
│   Agent Loop      │
│                   │
│  1. 接收问题       │
│  2. 调用 Tool      │────▶ query_database(sql)
│  3. 解释结果       │◀──── { rows: [...] }
│  4. 回复用户       │
└───────────────────┘
```

## 第 1 步：准备数据库

```typescript
import Database from 'better-sqlite3';

const db = new Database(':memory:'); // 内存数据库，演示用

// 创建示例表
db.exec(`
  CREATE TABLE employees (
    id INTEGER PRIMARY KEY,
    name TEXT NOT NULL,
    department TEXT NOT NULL,
    position TEXT NOT NULL,
    salary INTEGER NOT NULL,

  INSERT INTO employees VALUES

```

## 第 2 步：定义查询 Tool

```typescript
const queryDatabase = {
  name: 'query_database',
  description: `在 SQLite 数据库上执行 SELECT 查询。
  数据库包含 employees 表，字段有：id, name, department, position, salary, hire_date。
  仅支持 SELECT 查询，不支持 INSERT/UPDATE/DELETE。
  返回查询结果的 JSON 数组。`,
  parameters: {
    type: 'object',
    properties: {
      sql: {
        type: 'string',
        description: '要执行的 SELECT 语句。必须使用正确的表名和字段名。',
      },
    },
    required: ['sql'],
  },
  handler: async ({ sql }: { sql: string }) => {
    // 安全检查：只允许 SELECT
    const trimmed = sql.trim().toUpperCase();
    if (!trimmed.startsWith('SELECT')) {
      return { error: '仅支持 SELECT 查询', sql };
    }

    try {
      const rows = db.prepare(sql).all();
      return { success: true, rows, count: rows.length };
    } catch (error: any) {
      return {
        success: false,
        error: error.message,
        sql,
        hint: '请检查 SQL 语法。表名：employees，字段：id, name, department, position, salary, hire_date',
      };
    }
  },
};
```

## 第 3 步：Agent 循环

参考 [mcp-agent](https://github.com/lastmile-ai/mcp-agent)（8.3k stars）提供的 Agent 循环模式：

```typescript
import { Agent } from '@anthropic-ai/sdk';

async function dataQueryAgent(userQuestion: string): Promise<string> {
  const agent = new Agent({
    model: 'claude-sonnet-4-6',
    systemPrompt: `你是数据分析助手。用中文回答。当用户询问数据相关问题时：
1. 使用 query_database tool 查询数据库
2. 分析查询结果
3. 用自然语言解释给用户

数据库结构：
- employees 表：id, name, department, position, salary, hire_date`,
    tools: [queryDatabase],
  });

  const response = await agent.sendMessage(userQuestion);
  return response;
}

// 测试
console.log(await dataQueryAgent('工程部有哪些员工？'));
// → "工程部共有 3 位员工：张三（高级工程师）、王五（初级工程师）和陈七（架构师）。"
```

## 第 4 步：高级 —— 多轮查询

真实场景中，用户可能会追问：

```typescript
async function interactiveQuery() {
  const agent = new Agent({
    model: 'claude-sonnet-4-6',
    systemPrompt: '你是数据分析助手，支持多轮对话。记录上下文并支持追问。',
    tools: [queryDatabase],
  });

  // 第一问
  console.log(await agent.sendMessage('公司有多少员工？'));
  // → "公司共有 5 位员工。"

  // 追问（Agent 记得上下文）
  console.log(await agent.sendMessage('其中工程部薪资最高的是谁？'));
  // → Agent 自动查工程部员工 → 按薪资排序 → "陈七，架构师，月薪 75,000。"
}
```

## 错误处理案例

Agent 最常见的问题：**生成的 SQL 有语法错误**。

```typescript
// 用户问："工程人有多少？"（打错字）
// Agent 可能生成：SELECT COUNT(*) FROM employees WHERE department = '工程人'
// → 返回 0 条结果

// 更健壮的 Tool Handler：
const queryDatabaseRobust = {
  ...queryDatabase,
  handler: async ({ sql }) => {
    try {
      const rows = db.prepare(sql).all();
      if (rows.length === 0) {
        // 返回所有部门名帮助 Agent 纠错
        const depts = db.prepare('SELECT DISTINCT department FROM employees').all();
        return {
          success: true,
          rows: [],
          count: 0,
          hint: `没有匹配结果。可用的部门有：${depts.map((d: any) => d.department).join('、')}`,
        };
      }
      return { success: true, rows, count: rows.length };
    } catch (error: any) {
      return { success: false, error: error.message };
    }
  },
};
```

## 延伸：Text-to-SQL 最佳实践

1. **给 LLM 完整的表结构信息**：在 system prompt 或 tool description 中描述所有表和字段
2. **限制权限**：只允许 SELECT，绝不允许 DROP/ALTER
3. **提供纠错反馈**：SQL 语法错误时返回具体错误信息，让 Agent 自动修复
4. **样本数据提示**：在 prompt 中放几条示例数据，帮助 LLM 理解数据分布
