# 搭建 MCP Server

> 💡 **让任何 API 都变成 Agent 的工具** — 写一次 MCP Server，所有 MCP 兼容的 Agent 都能用。

## 第 2 步：创建第一个 MCP Server

```python
# server.py
from fastmcp import FastMCP
import httpx

mcp = FastMCP("Weather Server", version="1.0.0")

@mcp.tool()
async def get_weather(city: str, unit: str = "celsius") -> str:
    """获取指定城市的实时天气信息。

    当用户询问当前天气、温度、是否下雨等问题时使用。
    
    Args:
        city: 城市名称，如 "北京"、"上海"
        unit: 温度单位，celsius(摄氏度) 或 fahrenheit(华氏度)
    """
    # 使用免费的天气 API
    async with httpx.AsyncClient() as client:
        resp = await client.get(
            "https://wttr.in/" + city,
            params={"format": "%C %t %h %w", "lang": "zh"}
        )
        return f"{city}天气：{resp.text.strip()}"

@mcp.tool()
async def send_email(to: str, subject: str, body: str) -> str:
    """发送邮件。当用户说要"发邮件"、"通知某人"时使用。
    
    Args:
        to: 收件人邮箱地址
        subject: 邮件主题
        body: 邮件正文内容
    """
    # 模拟发送（真实场景接入 SendGrid/Resend API）
    return f"邮件已发送：\n收件人: {to}\n主题: {subject}\n---\n{body}"

if __name__ == "__main__":
    # stdio 模式：供 Claude Desktop 等本地使用
    mcp.run()
```

## 第 3 步：测试 MCP Server

### 方式 A：用 MCP Inspector 调试

```bash

```

打开浏览器，可以看到注册的 Tool 列表，手动测试。


重启 Claude Desktop → 你的 Tool 出现在 Agent 工具列表中。

```python

    mcp.run(transport="http", host="0.0.0.0", port=3001)
```

```bash
python server.py
# → MCP Server running on http://localhost:3001
```

## 第 4 步：添加更多 Tool

```python
@mcp.tool()
async def search_web(query: str, max_results: int = 5) -> str:
    """搜索网页内容。当用户询问最新信息、新闻、或需要查资料时使用。
    
    Args:
        query: 搜索关键词
        max_results: 最大结果数，默认 5
    """
    # 接入搜索 API（如 Tavily、Brave Search）
    results = await search_api(query, max_results)
    return "\n\n".join(
        f"{i+1}. {r['title']}\n   {r['url']}\n   {r['snippet']}"
        for i, r in enumerate(results)
    )

@mcp.tool()
async def read_file(path: str) -> str:
    """读取本地文件内容。当用户说"打开XX文件"、"看看XX文档"时使用。
    
    Args:
        path: 文件的绝对路径
    """
    try:
        with open(path, 'r', encoding='utf-8') as f:
            content = f.read()
        return f"文件: {path}\n---\n{content[:5000]}"  # 限制长度
    except Exception as e:
        return f"读取失败: {str(e)}"
```

## 常见问题

### Q：Tool 的 description 写多详细？

A：越详细越好。description 是 LLM 决定"要不要用这个 Tool"的唯一依据。

### Q：一个 MCP Server 放几个 Tool？

A：5-10 个为佳。太多会降低选择准确率。可以按领域拆分多个 MCP Server。

### Q：怎样处理 Tool 执行错误？

A：不要抛异常。返回结构化的错误信息，让 Agent 能理解并重试。

```python
@mcp.tool()
async def query_database(sql: str) -> dict:
    try:
        rows = execute_sql(sql)
        return {"success": True, "data": rows}
    except Exception as e:
        return {
            "success": False,
            "error": str(e),
            "hint": "可用的表：users, orders, products"
        }
```
