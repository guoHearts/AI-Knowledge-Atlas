# Self-healing 模式——自动纠错

> 💡 **让 Agent 自己修 Bug** — 生成 → 执行 → 观察错误 → 自动修正 → 重新执行。

## 动机

LLM 生成的代码/决策不可能一次就完美。Self-healing 通过"反馈循环"自动提高成功率。

```
传统方式（人工介入）：
Agent 生成代码 → 执行报错 → 你复制错误 → 告诉 Agent → Agent 修改 → 你再执行...

Self-healing 方式（自动化）：
Agent 生成代码 → 执行报错 → Agent 自动读取错误 → Agent 自己修改 → Agent 重新执行
                                                              ↑_____________|
                                                              最多重试 3 次
```

## 结构

```
     ┌─────────┐      ┌─────────┐      ┌──────────┐
     │ Generate │─────▶│ Execute │─────▶│ ✅ 成功  │
     └─────────┘      └─────────┘      └──────────┘
                            │
                       ❌ 失败 │
                            ▼
                      ┌──────────┐
                      │ Analyze  │  读取错误信息
                      └──────────┘
                            │
                            ▼
                      ┌──────────┐
                      │   Fix    │  基于错误修改
                      └──────────┘
                            │
                            ▼
                      ┌──────────┐
                      │ Re-exec  │  重试（最多 N 次）
                      └──────────┘
```

## TypeScript 实现

```typescript
interface RetryContext {
  attempt: number;
  maxRetries: number;
  errors: string[];
  code: string;
}

async function selfHealingExecute(
  generateCode: (context?: string) => Promise<string>,
  executeCode: (code: string) => Promise<{ success: boolean; output?: string; error?: string }>,
  maxRetries = 3
): Promise<{ success: boolean; result: string; attempts: number; history: RetryContext[] }> {
  const history: RetryContext[] = [];
  let errorContext = '';

  for (let attempt = 0; attempt <= maxRetries; attempt++) {
    console.log(`[Self-healing] 尝试 ${attempt + 1}/${maxRetries + 1}`);

    // 生成（带错误上下文）
    const code = await generateCode(errorContext);

    // 执行
    const result = await executeCode(code);

    if (result.success) {
      return {
        success: true,
        result: result.output!,
        attempts: attempt + 1,
        history,
      };
    }

    // 记录失败信息
    const errorInfo = {
      attempt,
      maxRetries,
      errors: [...(history[attempt - 1]?.errors || []), result.error!],
      code,
    };
    history.push(errorInfo);

    // 构造错误上下文给下次生成
    errorContext = `
上一次生成的代码执行失败。
错误信息: ${result.error}
代码:
\`\`\`
${code}
\`\`\`

请分析错误原因并修复。`;
  }

  return {
    success: false,
    result: `超过最大重试次数 (${maxRetries})，历史错误: ${history.map(h => h.errors).flat().join('; ')}`,
    attempts: maxRetries + 1,
    history,
  };
}

// 使用示例
const result = await selfHealingExecute(
  // Generator: 基于错误上下文生成代码
  async (errorContext) => {
    const agent = new Agent({
      model: 'claude-sonnet-4-6',
      systemPrompt: `你是一个 TypeScript 开发者。生成可运行的代码。
${errorContext ? `\n${errorContext}\n请修复以上错误。` : ''}`,
    });
    return agent.sendMessage('写一个函数，计算斐波那契数列的第 N 项');
  },

  // Executor: 执行代码并返回结果
  async (code) => {
    try {
      const func = new Function(`return (${code})`)();
      const output = func(10);
      return { success: true, output: String(output) };
    } catch (error: any) {
      return { success: false, error: error.message };
    }
  },

  3  // 最多重试 3 次
);

console.log(`成功: ${result.success}, 尝试次数: ${result.attempts}`);
```

## Python 实现

```python
import traceback

async def self_healing_loop(
    task: str,
    max_retries: int = 3
) -> str:
    """Agent 自我修复循环"""
    error_feedback = ""
    
    for attempt in range(max_retries + 1):
        print(f"[Try {attempt + 1}/{max_retries + 1}]")
        
        # Agent 生成代码
        from mcp_agent import Agent
        agent = Agent(
            model="claude-sonnet-4-6",
            instruction=f"你是 Python 开发者，生成可运行代码。{error_feedback}",
        )
        code = await agent.send_message(task)
        
        # 尝试执行
        try:
            exec_globals = {}
            exec(code, exec_globals)
            return f"✅ 成功 (尝试 {attempt + 1} 次)"
        except Exception as e:
            tb = traceback.format_exc()
            error_feedback = f"""
上次代码执行失败:
错误: {str(e)}
Traceback: {tb}

请分析错误并修复代码。重点关注:
1. 语法错误
2. 变量未定义
3. 类型错误
"""
    
    return f"❌ 失败：超过最大重试次数 {max_retries}"
```

## 防止死循环

```typescript
// 安全机制
class SelfHealingGuard {
  private seenErrors = new Set<string>();
  private readonly ERROR_SIMILARITY_THRESHOLD = 0.8;

  shouldRetry(error: string, attempt: number, maxRetries: number): {
    retry: boolean;
    reason?: string;
  } {
    // 检查 1：超过最大重试次数
    if (attempt >= maxRetries) {
      return { retry: false, reason: '超过最大重试次数' };
    }

    // 检查 2：同样的错误反复出现（死循环检测）
    const similarError = [...this.seenErrors].find(
      e => this.similarity(e, error) > this.ERROR_SIMILARITY_THRESHOLD
    );
    if (similarError) {
      return { retry: false, reason: '检测到死循环：同样的错误反复出现' };
    }

    this.seenErrors.add(error);
    return { retry: true };
  }

  private similarity(a: string, b: string): number {
    // 简单的 Jaccard 相似度（生产环境用更好的算法）
    const setA = new Set(a.toLowerCase().split(/\s+/));
    const setB = new Set(b.toLowerCase().split(/\s+/));
    const intersection = new Set([...setA].filter(x => setB.has(x)));
    return intersection.size / Math.max(setA.size, setB.size);
  }
}
```

## 取舍

| 该用时 | 不该用时 |
|--------|---------|
| 代码生成场景 | 错误不可自动修复 |
| 失败原因可从错误信息推断 | 修复成本 > 重写成本 |
| 成功率可通过重试提升 | 有副作用操作（扣款/发送） |
