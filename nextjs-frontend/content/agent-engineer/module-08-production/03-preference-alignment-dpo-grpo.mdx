# 偏好对齐——DPO 与 GRPO

> 💡 **让模型的输出更符合人类偏好** — 不只是"正确"，还要"有用、安全、有礼貌"。

## 从 RLHF 到 DPO

```
传统 RLHF（三步，复杂）：
1. SFT（监督微调）
2. 训练 Reward Model（奖励模型，需要大量人工标注）
3. PPO 强化学习（不稳定，难调参）

DPO（一步到位）：
1. 准备偏好数据对：好回答 vs 坏回答
2. 直接优化，不需要 Reward Model
```

## DPO 数据格式

```python
# DPO 数据：每条包含一个 prompt + 好回答 + 坏回答
dpo_data = [
    {
        "prompt": "如何学习编程？",
        "chosen": "建议从 Python 开始，因为语法简单。推荐资源：Python 官方教程、freeCodeCamp。每天坚持练习 1 小时，3 个月可入门。",  # 好的
        "rejected": "随便学，看心情。",  # 差的
    },
    {
        "prompt": "帮我写一封请假邮件",
        "chosen": "尊敬的领导：\n您好！因身体不适，需要请假一天（X月X日）。相关工作已安排好。\n谢谢！",  # 好的
        "rejected": "我明天不来。",  # 差的
    },
]
```

## DPO 实战（TRL）

```python
from transformers import AutoModelForCausalLM, AutoTokenizer
from trl import DPOTrainer, DPOConfig
from datasets import Dataset

# 加载模型
model = AutoModelForCausalLM.from_pretrained("mistralai/Mistral-7B-v0.1")
tokenizer = AutoTokenizer.from_pretrained("mistralai/Mistral-7B-v0.1")
tokenizer.pad_token = tokenizer.eos_token

# 准备数据集
dataset = Dataset.from_list(dpo_data)

# DPO 配置
dpo_config = DPOConfig(
    output_dir="./dpo-output",
    per_device_train_batch_size=2,
    learning_rate=5e-5,
    max_steps=200,
    beta=0.1,          # DPO 温度参数：越小 → 越贴近偏好
    logging_steps=10,
)

# 训练
trainer = DPOTrainer(
    model=model,
    tokenizer=tokenizer,
    train_dataset=dataset,
    args=dpo_config,
)

trainer.train()
```

## GRPO —— 推理增强

GRPO (Group Relative Policy Optimization) 是 DeepSeek-R1 使用的技术：

```
GRPO 流程：
1. 对同一个问题，模型生成 8 个不同回答
2. 用规则打分（如：数学题看答案对不对，代码看能不能跑通）
3. 好的回答加分，差的回答减分
4. 用这 8 个回答的"相对好坏"来更新模型

优势：
- 不需要人类标注偏好数据
- 用可自动验证的规则（数学、代码）就能提升推理能力
```

```python
# GRPO 概念代码（简化版）
def grpo_step(model, prompts, reward_function, group_size=8):
    for prompt in prompts:
        # 生成 group_size 个回答
        responses = [model.generate(prompt) for _ in range(group_size)]
        
        # 自动打分
        scores = [reward_function(r) for r in responses]
        
        # 计算相对优势（组内归一化）
        mean_score = np.mean(scores)
        std_score = np.std(scores) + 1e-8
        advantages = [(s - mean_score) / std_score for s in scores]
        
        # 好回答强化，差回答弱化
        for response, advantage in zip(responses, advantages):
            loss = -advantage * model.log_prob(response)
            model.backward(loss)
```

## DPO vs GRPO 对比

| 维度 | DPO | GRPO |
|------|-----|------|
| **数据需求** | 偏好对（人工标注） | 规则（自动验证） |
| **适用场景** | 风格/安全对齐 | 推理能力增强 |
| **代表模型** | Zephyr | DeepSeek-R1 |
| **成本** | 标注成本 | 算力成本（8x 生成） |
